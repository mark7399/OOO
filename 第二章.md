## 一句话版提示词（最短但有效）

> 用 ManimCE 写一个“发牌式”动画：所有分母元素（n、上标2、+、⋯）都从分子 n² 的同一发射点 TransformFromCopy 飞过去，**从右到左依次填满**。分母排版**不要用 MathTex 自动整句排版**，而是：每个目标元素的 x 必须取分子对应 glyph 的 `get_x()`，y 统一为 `denom_y`（由 dummy_denom 定）。动画节奏：每一项 `n` 和 `2` 同时飞到位，符号并入该项（或稍滞后），整串用 `LaggedStart` 逐项触发，不能同时到达。

## 结构化提示词（推荐：最容易一次生成对的代码）

你可以这样发给 AI（把你的对象名/索引换成你项目里的）：

**Prompt 模板：**

1. **排版规则（强约束）**
    

- 不要 `denominator2 = MathTex("n^2 + ...")` 来排版分母整句。
    
- 必须“手工构造分母每个元素”为独立 mobject，并强制对齐：
    
    - 数字位（分子 `numerator[0]` 的索引 `[0,2,4,6]`）对应目标 `n^2`：  
        `target_n.set_x(numerator[0][i].get_x())`，`target_n.set_y(denom_y)`  
        `target_2.move_to(target_n.get_corner(UR)+...)`
        
    - 符号位（分子 `numerator[0]` 的其它索引）对应目标 `+` 或 `\cdots`：  
        `target_sign = numerator[0][i].copy(); target_sign.set_y(denom_y)`
        
- 字号必须一致：目标 n/+ /⋯ 用 `copy()` 自分子；目标 2 用 `exp2_new[-1].copy()`。
    

2. **发射点规则（强约束）**
    

- 创建一个 emitter 放在分子 n² 位置，包含 `emitter_n, emitter_2, emitter_plus, emitter_dots`。
    
- 所有 `TransformFromCopy` 的 source 一律来自 emitter（不要从分子各处取）。
    

3. **动画节奏规则（强约束）**
    

- 不能把所有 TransformFromCopy 放同一个 `play` 里让它们一起结束。
    
- 必须按“项（term）”分组：
    
    - 每一项：`AnimationGroup(n,2)` 同时飞到位；
        
    - 符号加入该项（`AnimationGroup(prev, sign)` 或 `Succession(n2_group, sign)` 做滞后）；
        
- 整体用 `LaggedStart(*term_groups, lag_ratio=...)`，并要求方向**从右到左**（遍历 `reversed(range(...))` 或先构建 list 再反转）。
    

4. **输出要求**
    

- 给出可直接替换的代码段，不要伪代码。
    
- 默认 manimce 版本不支持 tex_string/get_parts_by_tex（所以不要用它们），用索引判断：`dots` 是 `numerator[0][-2]`。

## 你要“精准”的关键字（让 AI 不走歪路）

以后你只要在提示里出现这些词，AI 出错率会明显下降：

- **“不要用 MathTex 自动排整句分母，必须按分子 glyph 的 x 坐标手工锚定”**
    
- **“目标对象必须 copy 自分子/已有指数对象以保证字号一致”**
    
- **“按 term 分组，LaggedStart 逐项触发，不能同时到达”**
    
- **“扑克发牌/流水线式：每项 n 和 2 同时飞，符号并入该项（可滞后）”**
    
- **“从右到左”**（明确写：`reversed(range(...))` 或 `sort by x descending`）

具体到 Manim，就是：

- **布局：**“copy glyph + set_x / set_y”
    
- **时序：**“term_groups + LaggedStart / Succession”
