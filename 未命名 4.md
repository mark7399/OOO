既然改变常数系数无法动摇序列‘匀速增长’的本质，那该怎样才能改变一个序列的**求积系数**呢？

答案是：只需要让自然数列不再是简单的等差数列即可。

让我们提高每一项的幂次。比如将其提升为二次，得到一个平方数序列。此时，我们再次取最大项作为基准进行比较。你会发现，随着项数的增加，这个比值越来越接近 1/3。由于我们研究的是连续且光滑的图形，具体的项数不再重要，必然要取到无穷项。因此，沃利斯直接确立了结论：二次幂序列的**求积系数**是 1/3。

以此类推，能够得到 $n$ 次幂的**求积系数**就是 $\frac{1}{n+1}$。

不仅如此，沃利斯十分自然地认为 $n$ 可以取分数甚至负数（即便这个公式最初仅仅是从正整数归纳出来的）。

我们可以这样去理解他的逻辑跨越：

比如原本的序列：$0+1+2+3+\cdots$

按照公式，它的指数为 1，对应的**求积系数**是 1/2。

而在代数规律中，一次幂可以表示为两个二分之一次幂相乘，即：

$0^{(1/2)} \cdot 0^{(1/2)} + 1^{(1/2)} \cdot 1^{(1/2)} + 2^{(1/2)} \cdot 2^{(1/2)} + \cdots$

既然公式可以写成 $\frac{1}{1/2 + 1/2 + 1} = \frac{1}{2}$，那么顺应这种代数结构的对称性，一个单纯由二分之一次幂组成的序列（即各项开平方），它的**求积系数**，理所当然就是 $\frac{1}{1/2 + 1}$，也就是 2/3。”

1. **源对象隔离 (Source Isolation)**：

- 使用 `MathTex` 构建公式时，必须将“作为发牌源头的元素”作为独立的字符串参数传入。

- 示例：`source_tex = MathTex(r"前置公式...", r"核心元素", r"...后置公式")`，以便通过索引 `source_tex[1]` 独立调用。

2. **目标阵列提取 (Target Grouping)**：

- 构建目标序列（如分母）后，必须使用列表推导式或切片，将需要飞入的“坑位”元素单独提取成一个 `VGroup`。

- 示例：`targets = VGroup(*[target_tex[i] for i in 目标索引列表])`。

3. **构建动画列表 (Animation Array)**：

- 必须使用列表推导式结合 `TransformFromCopy`，生成从源元素到所有目标元素的动画列表。

- 示例：`animations = [TransformFromCopy(source_tex[1], t) for t in targets]`。

4. **错位播放 (Staggered Playback)**：

- 在 `self.play` 中使用 `*` 解包动画列表。

- **核心重点**：必须添加 `lag_ratio=0.1` (或 0.05~0.2 之间的小数) 参数，利用时间差制造出发牌式的丝滑视觉效果。

- 示例：`self.play(*animations, run_time=2, lag_ratio=0.1)`。

“沃利斯研究的核心对象是序列求和，也就是级数。

比如这三个序列和： 
0 + 1 + 2 + 3 + ··· 
0 + 0.1 + 0.2 + 0.3 + ··· 
0 + 2 + 4 + 6 + ···

虽然它们值累加的速度各不相同，但在沃利斯眼中，它们全都是一回事。因为它们都是由自然数列乘以一个特定的系数生成的。

**改变系数只是改变了它们每次跨步的大小，而丝毫没有改变它们‘匀速增长’的本质。**
